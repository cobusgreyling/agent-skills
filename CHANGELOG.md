# Changelog

All notable changes to this collection are documented here.

The format is loosely based on [Keep a Changelog](https://keepachangelog.com/). The repo follows semver-ish versioning at the collection level: a **minor** bump when skills are added, a **patch** bump when an existing skill changes.

## [Unreleased]

### Added
- **Nine new skills**, each with `SKILL.md`, `EXAMPLES.md`, `TRANSCRIPT.md`:
  - `memory-design` — working / episodic / semantic / procedural; recall budgets, TTLs, conflict resolution.
  - `multi-agent-orchestration` — handoff test, typed payloads, supervisor patterns, refusing peer-to-peer chat.
  - `model-routing` — per-call tier selection, cheap-first + escalate, capability gating, draft-then-verify.
  - `prompt-injection-defense` — threat model, indirect injection, two-model isolation, action authorisation.
  - `structured-output-reliability` — function calling vs JSON mode vs structured outputs; strict validation; no regex repair.
  - `tool-failure-handling` — classification (read-safe / idempotent / non-idempotent / irreversible), idempotency keys, reconcile-on-timeout, compensation.
  - `llm-as-judge` — pairwise vs pointwise, position bias, length bias, self-preference, meta-eval against humans.
  - `human-in-the-loop` — tiered actions, out-of-band confirmation, decision-UI design, approval-fatigue probes.
  - `context-window-hygiene` — cache alignment, tool-result summarisation, fixed-boundary compaction, refusing "bigger window" as the answer.
- **Three additional 20-task evals** under `eval/`:
  - `agent-architecture-patterns/`
  - `agent-cost-modeling/`
  - `guardrails-and-safety/`
  - Brings eval coverage from 2 of 9 skills to 5 of 18.
- `AUTHORING.md` — meta-skill for writing in the house style (hard line, anti-patterns, decision flow, eight conventions).
- `FAILURES.md` — failure-pattern → skill index. Browse by symptom; click through to the brief that catches it.
- `scripts/convert.py` — auto-generates per-agent variants (Cursor MDC, Codex AGENTS.md, Gemini @-refs) from the canonical SKILL.md. Removes manual drift.
- `scripts/transcript_regression.py` — replays each TRANSCRIPT.md against current models; flags skills where the baseline has drifted toward the with-skill expectation.
- Cross-agent install matrix in `README.md` (Claude Code native, plus Gemini CLI, Cursor, Codex loading instructions).
- "What makes this different" section in `README.md` — surfaces the production-triage / hard-line angle.
- `Related:` cross-link blocks at the bottom of every `SKILL.md`. Skills now point at the next decision along the path.
- `skills.json` — machine-readable manifest at the repo root. Generated by `scripts/build_index.py`.
- `TRANSCRIPT.md` for every skill — one realistic prompt, agent-without-skill response, agent-with-skill response, annotated diff.
- `eval/tool-use-schema-design/` — reproducible 20-task eval with prompts, rubric, scoring script, and a results template.

### Changed
- `README.md` install section rewritten end-to-end. Drops the single-Claude path; ships an explicit compatibility matrix.
- `README.md` adds a `Tooling` section and a pointer to `AUTHORING.md` and `FAILURES.md`.
- Eval coverage statement updated from "two skills" to "five skills".
